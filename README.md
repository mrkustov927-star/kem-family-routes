# Семейные маршруты по Кеми с Первыми

**[Открыть интерактивный маршрут](https://mrkustov927-star.github.io/kem-family-routes/)**

Мобильный веб-помощник для юных экскурсоводов: интерактивная карта, карточки точек, режим проведения экскурсии и сбор предложений правок.

## Что есть в прототипе

- десять подготовленных точек маршрута по Кеми;
- полный редакционный план из 41 темы с делением на пять глав;
- статусы готовности и кнопка сбора материала для каждой будущей точки;
- современная Яндекс Карта с актуальной уличной подложкой, масштабированием, полноэкранным режимом, линией маршрута, кликабельными маркерами и фотослоем;
- переключатель на резервную карту CARTO/OpenStreetMap, если Яндекс Карта недоступна;
- сохранённый пешеходный черновик по уличной сети OpenStreetMap: 4,5 км и около 53 минут чистой ходьбы между десятью подготовленными точками;
- определение местоположения и расстояния до ближайшей готовой точки;
- режим полевой фиксации координат прямо на карте;
- добавление новых экскурсионных точек со своим названием, координатой и комментарием;
- режим полевой проверки пути: участники отмечают закрытые проходы, опасные переходы, отсутствие тротуара и другие проблемы прямо на красной линии;
- визуальный архив из оптимизированных фотографий с редакционными подписями;
- добавление участниками фотографий с телефона: выбор точки, подпись, авторство, подтверждение прав и локальный предпросмотр;
- отдельный режим «Провожу экскурсию»;
- подсказки: что показать, как рассказать, какой вопрос задать семье;
- отметка пройденных точек с сохранением на устройстве;
- форма предложений правок и единая выгрузка текстов, координат, замечаний и фотографий в JSON;
- замечания к маршруту сохраняются на устройстве, показываются отдельным слоем и входят в общую JSON-выгрузку;
- базовый офлайн-кэш интерфейса;
- автоматическая публикация на GitHub Pages после обновления `main`.

## Важные ограничения версии 0.3

- на карте полностью раскрыты десять точек; остальные перечислены в плане и ожидают исследования;
- жёлтые маркеры требуют полевой проверки координат;
- пешеходная линия автоматически рассчитана по открытым картографическим данным; переходы, тротуары и безопасные места остановок необходимо проверить на местности;
- опубликованные фотографии имеют редакционные пометки; авторство и даты части кадров ещё уточняются;
- предложения, новые точки и фотографии пока сохраняются только в браузере конкретного устройства. Для передачи редактору участник использует кнопку «Выгрузить правки»;
- фотографии автоматически уменьшаются перед локальным сохранением; большие оригиналы нужно хранить отдельно до принятия редактором;
- общая редакционная база с аккаунтами и синхронизацией между устройствами — следующий этап;
- для подложки карты требуется интернет.

Клиентский ключ JavaScript API Яндекс Карт используется только на публичной странице и ограничен доменом GitHub Pages в кабинете разработчика.

## Запуск локально

Откройте папку через любой статический веб-сервер, например:

```bash
python3 -m http.server 8080
```

После этого откройте `http://localhost:8080`.

## Редакционная модель следующего этапа

1. Участник предлагает правку или фотографию.
2. Редактор видит черновик и проверяет источник.
3. Материал получает статус «проверено».
4. Только после проверки изменение появляется у всех пользователей.

Так дети могут участвовать в наполнении проекта, а на публичной карте остаются проверенные формулировки.

## Источники

Ссылки на музейные, муниципальные и картографические материалы указаны непосредственно в карточках точек. Исторические формулировки необходимо дополнительно сверять с краеведами и архивными документами.
