# Подключение общей редакционной карты

Сайт уже содержит публичный URL проекта и publishable key. Осталось один раз создать таблицы и участников в Supabase.

## 1. Создать структуру базы

1. Откройте проект Supabase `mrkustov927-star's Project`.
2. В меню выберите **SQL Editor** → **New query**.
3. Откройте файл [`migrations/20260718190000_shared_map.sql`](./migrations/20260718190000_shared_map.sql), скопируйте его целиком в редактор и нажмите **Run**.
4. После успешного выполнения в **Table Editor** появятся таблицы `points`, `photos`, `routes`, `route_waypoints`, `profiles` и `point_revisions`.

Скрипт можно выполнить повторно: он обновляет правила доступа без удаления материалов.

## 2. Настроить ссылки входа

В разделе **Authentication** → **URL Configuration** укажите:

- Site URL: `https://mrkustov927-star.github.io/kem-family-routes/`
- Redirect URL: `https://mrkustov927-star.github.io/kem-family-routes/`

Это нужно для кнопки «Получить ссылку на почту».

## 3. Добавить участников

1. Откройте **Authentication** → **Users**.
2. Нажмите **Add user**.
3. Укажите почту и временный пароль участника.
4. Передайте данные только этому участнику и попросите сменить пароль при необходимости.

Публичной регистрации на сайте нет: новые учётные записи создаёт руководитель проекта.

## 4. Первая синхронизация

1. Откройте сайт и нажмите **Войти**.
2. Войдите под созданной учётной записью.
3. При первом входе сайт автоматически перенесёт в общую базу исходный список из 41 точки.

После этого зелёный индикатор «Общая карта» означает, что изменения синхронизируются между устройствами.

## Что могут участники

- редактировать название, координаты и полный материал любой точки;
- ставить новые точки на карте;
- загружать фотографии как общие фоточерновики;
- вручную прокладывать общую линию маршрута последовательными узлами по улицам;
- видеть изменения других участников без ручной выгрузки файлов.

Публичные посетители видят опубликованные материалы, но не могут изменять базу. Все изменения точек записываются в журнал `point_revisions`.
